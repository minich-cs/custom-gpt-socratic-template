# Contributing

Contributions are welcome, especially those that improve clarity, pedagogy, or adaptability across subjects.

This repository prioritizes educational intent over technical cleverness.

## How to Contribute

1. Fork the repository.
2. Make changes in your fork.
3. Commit changes with clear, descriptive messages.
4. Submit a pull request if you wish to propose changes upstream.

## What Makes a Good Contribution

Strong contributions typically:
- Improve prompt clarity or instructional tone
- Strengthen the Socratic, question-based approach
- Add helpful documentation or examples
- Make the project easier for educators to understand and adapt

## What to Avoid

Please avoid contributions that:
- Turn the system into an answer-delivery tool
- Remove teacher guidance or reflective elements
- Add unnecessary automation or complexity
- Include student data or assessment logic

## Explaining Changes

When proposing changes, briefly explain:
- What you changed
- Why you changed it
- How it supports inquiry-based learning

This context matters more than the code itself.

---

Thank you for helping keep this project thoughtful, usable, and grounded in good teaching practice.
